## Order of Operations

> These are paste-able prompts for the Claude for Mac desktop app (Code tab).
> Open the Code tab, paste the full contents of each prompt file, press Enter.

### Step 1 — Generate the roadmap

Paste the full contents of `PROMPT-1-roadmap.md` into the Code tab.

- Agent reads `vision.md`
- Writes `roadmap.md` with all 44 widgets across 3 phases
- Agent shows draft first — review and approve before it saves
- Do NOT proceed to Step 2 until `roadmap.md` has content

### Step 2 — Generate feature stubs

Paste the full contents of `PROMPT-2-stubs.md` into the Code tab.

- Agent reads `roadmap.md`
- Writes 44 stub files into `features/`
- Runs verification at the end (expected: 44 / 6 / 44)
- Do NOT proceed to Step 3 until all 3 counts pass

### Step 3 — Start building

Paste the contents of `~/auto-sdd/.claude/commands/build-next.md` into the Code tab.

- Picks first pending feature from `roadmap.md` (F-001: CRE Property Map)
- Generates spec, writes tests, implements, commits
- Repeat until all 44 features are ✅

## After Every Merge — Required Step

After merging any agent branch to main, you MUST push before running the next prompt:

```bash
cd ~/auto-sdd
git push origin main
```

The agent's repo sees `origin/main`. If you merge locally but don't push, the next prompt's precondition check (`git fetch origin` + `git log --oneline origin/main..main`) will find local main ahead of origin and STOP. This is the correct behavior — push first, then run the next prompt.

## Rules the Build Agent Must Follow

1. **No invented enumerations** — market names, building classes, space types, etc. must come from the canonical constants in each stub file
2. **No mock API endpoints in Phase 1** — use DuckDB queries against the real CSVs
3. **Submarket is never a static dropdown** — always loads dynamically filtered by selected Market
4. **Sales-side type filters** (Sale Type, Buyer Type, Seller Type) are text search inputs, never dropdowns
5. **Properties CSV coordinate format** is `LATITUDE` + `LONGITUDE` (separate columns), not `Geo Point` like leases/sales
6. **Widget-level filters override deck-level filters** when set by the user

## If Something Is Wrong

- `vision.md` is wrong → edit it directly, then re-run `/sitdeck-roadmap create`
- `roadmap.md` is wrong → re-run `/sitdeck-roadmap create` (force mode)
- A stub file is wrong → edit it directly, no need to regenerate all 44
- Build loop is stuck → check `roadmap.md` for 🔄 in-progress items, resolve manually
