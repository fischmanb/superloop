## Recommended Agent Split: 2 Sequential Agents

### Agent 4a — Support modules
Input: Preamble, Utilities, Gates, Eval Feedback, Drift, Branch Mgmt, Prompt Builders (~1190 bash lines)
Output: build_gates.py, drift.py, branch_manager.py, prompt_builder.py + tests (~600-700 Python lines)

### Agent 4b — Core orchestration
Input: Core Build Loop, Post-Loop, Main Body (~1062 bash lines) + 4a's actual output
Output: build_loop.py (BuildLoop class + main entry point) + tests (~500-600 Python lines)

### Why sequential not parallel
4b imports from 4a's modules. 4b needs to see 4a's actual output to wire imports — convention stubs aren't enough for internal modules.

### Why 2 not 1
2299 lines of bash + conventions doc + test output → ~15,000 input tokens before writing starts. Deduplication mandate (collapse 3 success paths) requires understanding whole flow, but utility modules are mechanical conversions.
